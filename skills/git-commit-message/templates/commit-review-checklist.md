# Commit review checklist

Use this checklist before accepting a draft commit message.

## Structural checks
- [ ] One logical change only
- [ ] Subject stands alone
- [ ] Blank line between subject and body when body exists
- [ ] Trailers appear only at the end

## Content checks
- [ ] Subject is specific
- [ ] Message matches repository convention
- [ ] Body explains why when needed
- [ ] No invented metadata
- [ ] No vague filler like `fix`, `stuff`, or `misc`

## Workflow checks
- [ ] Commit should not be split further
- [ ] Temporary review commits use `fixup!` or `squash!` when appropriate
- [ ] Breaking changes are explicit and confirmed
